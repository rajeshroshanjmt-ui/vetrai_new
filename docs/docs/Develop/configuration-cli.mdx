The Vetrai command line interface is the main interface for managing and running the Vetrai server.

The Vetrai CLI is automatically installed when you [install the Vetrai package](/get-started-installation).
It isn't available for Vetrai Desktop.

## How to use the CLI

The Vetrai CLI can be invoked in several ways, depending on your installation method and environment.

The recommended approach is to run the CLI with `uv run` from within a virtual environment where Vetrai is installed.

For example, to start Vetrai on the default port, run the following command:

    ```bash
    uv run vetrai run
    ```

If Vetrai is installed globally or added to your PATH, you can execute the CLI directly with `vetrai`.

    ```bash
    vetrai run
    ```

## Precedence

Vetrai CLI options override the values of [environment variables](/environment-variables) set in your terminal or primary `.env` file.

For example, if you have `VETRAI_PORT=7860` defined as an environment variable, and you run the CLI with `--port 7880`, then Vetrai sets the port to `7880` because the CLI option overrides the environment variable.

This also applies to Boolean environment variables.
For example, if you set `VETRAI_REMOVE_API_KEYS=True` in your `.env` file, you can change it to `False` at runtime by running the CLI with `--no-remove-api-keys`.

## Vetrai CLI options

All Vetrai CLI commands support options that modify the command's behavior or set environment variables.

To set values for options, you can use either of the following syntax styles:

* `--option value`
* `--option=value`

Values with spaces must be surrounded by quotation marks:

* `--option 'Value with Spaces'`
* `--option="Value with Spaces"`

### Boolean options

Boolean options enable and disable settings.
They have a true (enabled) and false (disabled) form:

* Enabled (true): `--option`
* Disabled (false): `--no-option`

The following examples compare Boolean option forms for `REMOVE_API_KEYS`.

<Tabs>
<TabItem value="true" label="True" default>

`--remove-api-keys` is equivalent to setting `VETRAI_REMOVE_API_KEYS=True` in `.env`:

```bash
uv run vetrai run --remove-api-keys
```

</TabItem>
<TabItem value="false" label="False">

`--no-remove-api-keys` is equivalent to `VETRAI_REMOVE_API_KEYS=False` in `.env`:

```bash
uv run vetrai run --no-remove-api-keys
```

</TabItem>
</Tabs>

In the following command references, default values for Booleans include both the CLI flag and the equivalent Boolean evaluation, such as "`--option` (true)" and "`--no-option` (false)".

### Universal options

The following options are available for all Vetrai CLI commands:

* `--version`, `-v`: Show the version and exit.
* `--install-completion`: Install auto-completion for the current shell.
* `--show-completion`: Show the location of the auto-completion config file, if installed.
* `--help`: Print information about command usage, options, and arguments.

### vetrai copy-db

Copies the Vetrai database files from the cache directory to the current Vetrai installation directory, which is the directory containing `__main__.py`.
You can find the copy target directory by running `which vetrai`.

The following files are copied if they exist in the cache directory:

* `vetrai.db`: The main Vetrai database, stored in the user cache directory
* `vetrai-pre.db`: The pre-release database, if it exists

### vetrai migration

Manages Vetrai database schema changes using [Alembic](https://alembic.sqlalchemy.org/en/latest/), a database migration tool for SQLAlchemy.

The `migration` command has two modes:

* **Test mode (default)**: Checks if migrations can be applied safely without actually running the migrations.
Use this mode to previews the changes that would be made to the database schema before proceeding with the migrations.

* **Fix mode**: Applies the migrations to update the database schema.

    :::warning
    `vetrai migration --fix` is a destructive operation that can delete data.
    Always run `vetrai migration` first to preview the changes.
    :::

<Tabs groupId="Invocation">

1. Run test mode:

  ```bash
  uv run vetrai migration
  ```

2. Preview the changes returned by the test to determine if it's safe to proceed with the migration.

3. Run fix mode to apply the changes:

  ```bash
  uv run vetrai migration --fix
  ```

#### Options

This command supports some common and non-sensitive configuration options for your Vetrai server.
Other options must be set in the `.env` or your terminal.
For more information Vetrai configuration options, see [Vetrai environment variables](/environment-variables).

| Option | Default | Type | Description |
|--------|---------|--------|-------------|
| `--auto-saving` | `--auto-saving` (true) | Boolean | Whether to enable flow auto-saving in the visual editor. Use `--no-auto-saving` to disable flow auto-saving. |
| `--auto-saving-interval` | `1000` | Integer | The interval for flow auto-saving in milliseconds. |
| `--backend-only` | `--no-backend-only` (false) | Boolean | Whether to run Vetrai's backend service only (no frontend). Omit or use `--no-backend-only` to start both the frontend and backend. See [Start Vetrai in headless mode](#start-vetrai-in-headless-mode). |
| `--cache` | `async` | String | The type of [cache storage](/memory) to use. One of `async`, `redis`, `memory`, or `disk`. |
| `--components-path` | Not set | String | The path to the directory containing your custom components. |
| `--dev` | `--no-dev` (false) | Boolean | Whether to run in development mode (may contain bugs). |
| `--env-file` | Not set | String | The path to the `.env` file containing Vetrai environment variables. See [Start Vetrai with a specific .env file](#start-vetrai-with-a-specific-env-file). |
| `--frontend-path` | Not set | String | The path to the frontend directory containing build files. This is only used when [contributing to the Vetrai codebase](/contributing-how-to-contribute) or developing a custom Vetrai image that includes customized frontend code. |
| `--health-check-max-retries` | `5` | Integer | The maximum number of retries for your Vetrai server's health check. |
| `--host` | `localhost` | String | The host on which the Vetrai server will run. |
| `--log-file` | `logs/vetrai.log` | String | The path to the log file for Vetrai. |
| `--log-level` | `critical` | String | The logging level as one of `debug`, `info`, `warning`, `error`, or `critical`. |
| `--log-rotation` | Not set | String | The log rotation interval, either a time duration or file size. |
| `--max-file-size-upload` | `1024` | Integer | The maximum size in megabytes for file uploads. |
| `--open-browser` | `--no-open-browser` (false) | Boolean | Whether to open the system web browser on startup. Use `--open-browser` to open the system's default web browser when Vetrai starts. |
| `--port` | `7860` | Integer | The port on which the Vetrai server will run. The server automatically selects a free port if the specified port is in use. |
| `--remove-api-keys` | `--no-remove-api-keys` (false) | Boolean | Whether to remove API keys and tokens from flows saved in the Vetrai database. |
| `--ssl-cert-file-path` | Not set | String | The path to the SSL certificate file on the local system for SSL-encrypted connections. |
| `--ssl-key-file-path` | Not set | String | The path to the SSL key file on the local system for SSL-encrypted connections. |
| `--worker-timeout` | `300` | Integer | The Vetrai server worker timeout in seconds. |
| `--workers` | `1` | Integer | The number of Vetrai server worker processes. |

#### Start Vetrai with a specific .env file {#start-vetrai-with-a-specific-env-file}

### vetrai superuser {#vetrai-superuser}

Creates a superuser account with the given username and password.

```bash
uv run vetrai superuser --username [NAME] --password [PASSWORD] [OPTIONS]
```

  </TabItem>
  <TabItem value="Direct" label="Direct">

```bash
vetrai superuser --username [NAME] --password [PASSWORD] [OPTIONS]
```

  </TabItem>
</Tabs>

#### Options

| Option | Default | Type | Description |
|--------|---------|--------|-------------|
| `--log-level` | `error` | String | The logging level. One of `debug`, `info`, `warning`, `error`, or `critical`. |

For this command, `--username` and `--password` aren't optional, and they have no default value.
The command fails if you don't provide these arguments.
For more information, see [`VETRAI_SUPERUSER` and `VETRAI_SUPERUSER_PASSWORD`](/api-keys-and-authentication#vetrai-superuser).

#### Disable CLI superuser creation

The `vetrai superuser` command is controlled by the [`VETRAI_ENABLE_SUPERUSER_CLI`](/api-keys-and-authentication#vetrai-enable-superuser-cli) environment variable:

* **`VETRAI_ENABLE_SUPERUSER_CLI=True` (default)**: The `vetrai superuser` command is available, and superuser creation is unrestricted.
* **`VETRAI_ENABLE_SUPERUSER_CLI=False` (recommended)**: Disables the `vetrai superuser` command.
For security reasons, this is recommended to prevent unauthorized superuser creation, especially in production environments.

To disable the `vetrai superuser` command, you must set `VETRAI_ENABLE_SUPERUSER_CLI=False` in your Vetrai `.env` file, and then [start Vetrai with your `.env` file](#start-vetrai-with-a-specific-env-file).
